# 动能 + 目标看图 SOP（实战手册）· v0.1

> 当 BTC 快速上涨 / 下跌时，**怎么用动能能量柱 + 目标投影两张卡判断**：
>
> 1. 动能足不足
> 2. 还能涨到 / 跌到哪
> 3. 衰竭多少 · 何时该撤
>
> 父文档：[`MOMENTUM-PULSE.md`](MOMENTUM-PULSE.md)（设计真源）。
> 本文是「使用手册」，任何阈值与设计文档冲突时以 `MOMENTUM-PULSE.md` 为准。

---

## 0. 阅读顺序（小白 30 秒）

```
① 动能卡顶部白话条 / 场景标签   → 方向 + 强度 + 风险
② 疲劳 chip                    → 还能烧多久
③ ⚡ override                   → 最近发生了什么大事
④ 目标卡 🎯/🛡 第一目标         → 涨/跌到哪、还有多远
⑤ 多 TF 三色灯带                → 多周期共振判断
```

**只看前 4 步就能下决策**。第 5 步给老手做"反向 / 逆趋势警惕"用。

---

## 1. 上涨场景 · 5 步看图

### Step 1：动能卡顶部白话条 → 3 秒判方向 + 强度

| 看到 | 含义 | 行动建议 |
|------|------|----------|
| 🟢 **多头强**（多 ≥ 60 vs 空 < 20） | 真上涨，资金集中烧多 | **顺势** |
| 🟢 **多头中等**（多 30~60） | 涨成立，但需事件确认 | 顺势小仓 / 等回踩 |
| 🟢 **多头弱**（多 < 30） | 看似涨实则虚弱 | 不追，等下根 |
| ⚪ **多空拉锯** + 价在涨 | 价涨动能不偏多 | **假突破嫌疑高** |
| 🔴 **空头中等以上** + 价**还在涨** | cvd / imbalance 反向 | **强烈警惕逼空陷阱** |

### Step 2：疲劳 chip → 还能烧多久

| Chip | 含义 |
|------|------|
| 🟢 **新鲜** | 油箱还满，可持续 |
| 🟡 **中段疲劳** | `trend_saturation ≥ 50%`，已烧一半，confidence ×0.8 |
| 🔴 **趋势衰竭** | exhaustion 报警 + 方向匹配，confidence ×0.5 → **随时反转** |

> 上涨中看到 🔴 衰竭 = 顶部嫌疑，**该止盈不该追多**。

### Step 3：⚡ override 事件 → 最近发生了什么大事

| 事件 | 含义 | 上涨中的解读 |
|------|------|--------------|
| ⚡ **Pierce↑** | ATR×N 足够大 → 真穿阻力 | **确认信号** |
| ⚡ **CHoCH↑ / BOS↑** | 机构结构突破 | **强确认** |
| ⚡ **Sweep↑** | 上方流动性被扫 | **反转嫌疑** |
| ⚡ **CHoCH↓ / Sweep↓** 在上涨中 | 反向事件 | **顶部警告** |

> 铁律：⚡ 是**事件**不是趋势。看到 ⚡↑ 但 score 偏空（拉锯/逼空场景）时，
> 事件**不构成方向**，仅作"刚才发生过"信号。

### Step 4：🎯 涨到 → 涨到哪里、还有多远

```
🎯 涨到 81,069   +0.5% · 约 6 根 · ⭐⭐⭐⭐
```

| 字段 | 看什么 |
|------|--------|
| 价格 | 第一磁吸位（最近 + 最高置信度自动选出） |
| 距离 % | 剩余空间 |
| 约 N 根 | 按当前 ATR 估的到达节奏（30m K 线 ≈ 1 根） |
| ⭐ 数 | 4★+ = 高置信，3★ 仅参考 |
| 透明度 | 越淡 = confidence 越低（被疲劳/距离折扣） |

下面列表给完整地图（T1/T2 多目标），按距离排：

| 图标 | 含义 | 上涨中怎么用 |
|------|------|--------------|
| 🎯 **ROI** | 主力赚钱目标 | **强多头目标**，首选 |
| 🛡 **Pain** | 多头能承受的最大回撤 | 在上方 = 反向止盈线 |
| 💣 **Cascade** | 大资金爆仓带 | 上方爆仓 = **诱多燃料**，警惕针刺 |
| 🌡 **Heatmap** | 清算密集线 | 强磁吸（清算盘吸价格） |
| 💨 **Vacuum** | 流动性真空区 | 价格**会快速穿越**（无支撑） |
| ◯ **Nearest** | 最近 R/S | 兜底参考 |

### Step 5：多 TF 灯带 → 多周期共振判断

| 灯带 | 含义 | 决策 |
|------|------|------|
| 🟢🟢🟢 全绿 | 30m / 1h / 4h 共振多头 | **最强多头**，可重仓 |
| 🟢🟢⚪ | 短中期多头，4h 中性 | 反弹/趋势启动初期 |
| 🟢⚪🔴 | 30m 绿 / 4h 红 | **逆大趋势反弹**，仓位收紧 |
| 🟢🔴🔴 | 30m 绿 / 1h+4h 红 | **典型逼空陷阱** |

## 2. 下跌场景 · 完全对称

把上面表里的「绿/多/涨/Pierce↑/🎯」全部换成「红/空/跌/Pierce↓/🛡」即可。

特别警惕的对称信号：

| 下跌中看到 | 解读 |
|-----------|------|
| 🔴 空头中等 + ⚡ **Sweep↓** | 下方流动性扫荡 → **底部反转嫌疑** |
| 🔴 空头 + 🔴 衰竭 | 跌到底部 → **该止盈空头** |
| 🛡 跌到 X · ⭐⭐⭐⭐ | 下方第一防线 → 多头会接 |
| 💣 在下方 | 多头爆仓燃料 → **针刺多反弹** |
| 💨 Vacuum 在下方 | 真空区 → **会快速插针** |

---

## 3. 终止 / 反转信号（必须撤退）

任何**一项**触发就要警惕：

1. **价格已撞 ⭐⭐⭐⭐+ 的目标** → 进入超买/超卖区，回撤概率高
2. **方向反向 ⚡ CHoCH / Sweep** → 立即警觉
3. **多 TF 灯带由绿转红**（或反向）→ 短周期已结束
4. **疲劳进入 🔴 exhausted** + 方向匹配 → 趋势已末段
5. **score 主导侧 强→中→弱→中性 的递降序列** → 动能在退潮

## 5. 自动场景识别（V1.1 · Step 7.5）

为减少人工对照，后端实现 `MomentumScenarioView`（参见 §2 触发表对应的 12 类）：

| Scenario ID | 触发条件 | 白话文案 | 风险等级 |
|------|------|------|------|
| `bull_a_grade` | 多 ≥ 60 · 空 < 20 · 新鲜 · 30m+1h 都偏多 | 🟢 A 级多头共振，趋势已确认 | low |
| `bear_a_grade` | 镜像 | 🔴 A 级空头共振，趋势已确认 | low |
| `bull_mid` | 多 30~60 · 主导=多 · 新鲜/中段疲劳 | 🟢 多头中等强度，等回踩 | mid |
| `bear_mid` | 镜像 | 🔴 空头中等强度，等反弹 | mid |
| `bull_late` | 主导=多 · 衰竭 · 价撞 ⭐⭐⭐⭐+ 上方目标 | 🟡 多头末段，进入止盈区 | high |
| `bear_late` | 镜像 | 🟡 空头末段，进入止盈区 | high |
| `short_squeeze` | 24h 价 ≥ +0.3% + 空头主导 + 1h 或 4h 灯带红 | ⚠ 逼空陷阱嫌疑，不追多 | high |
| `bull_trap` | 24h 价 ≤ -0.3% + 多头主导 + 1h 或 4h 灯带绿 | ⚠ 多头陷阱嫌疑，不追空 | high |
| `fake_breakout` | 证据链含 pierce 双向震荡 + 拉锯 | ⚠ 假突破震荡盘，观望 | mid |
| `top_reversal` | 上涨过程 ⚡ Sweep↓/CHoCH↓ | ⚠ 顶部反转嫌疑，止盈 | high |
| `bottom_reversal` | 下跌过程 ⚡ Sweep↑/CHoCH↑ | ✓ 底部反转嫌疑，可埋伏 | mid |
| `neutral` | 都不命中（fallback） | ⚪ 中性，无明显信号 | low |

匹配优先级（**风险高 → 风险低**）：

```
trap (短逼/多陷) > late (末段) > fake_breakout / reversal > a_grade > mid > neutral
```

> 高风险场景永远优先匹配，避免「真趋势」标签淹没「陷阱」警告。

## 6. 边界 / 常见误读

| 误读 | 真相 |
|------|------|
| 看到 ⚡ Pierce↑ 就追多 | Pierce 是**事件**，不等于趋势；要看 score 是否真偏多 |
| 看到 ⭐⭐⭐⭐⭐ 就以为必到 | confidence ≠ 概率；它是「来源 + 距离 + 对齐 + 疲劳」的加权 |
| 30m 绿就以为大趋势涨 | 必须看 1h / 4h 灯带是否同色，**单 TF 信号≠大趋势** |
| 多 + 空都低就以为没行情 | 可能是 **stale**（数据陈旧），看 freshness chip |
| 双向 Pierce 当成"双向突破强信号" | 这是**震荡假突破**，反而该观望 |
