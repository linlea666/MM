# 作战大屏 V1 设计方案

> **版本**：V1 MVP
> **目标**：在 5 秒内让用户回答 5 个问题 — 主力在干什么 / 市场什么阶段 / 关键位在哪 / 流动性往哪扫 / 现在该做什么
> **原则**：结论前置，指标让路；模块间有裁决；参与度门控可信度

## 二、中央结论带（🎯 Hero Strip）

屏幕最抢眼的一行。内容：

```
★★★★☆  阶段：真突破启动  |  主行为：吸筹  |  参与：主力真参与  |  动作：追多 @ 111050  |  失效：跌破 110300
```

### 字段来源

| 字段 | 产出模块 | 逻辑 |
|---|---|---|
| 星级 | 决策引擎 | `min(phase_confidence, participation_confidence) × action_confidence` |
| 阶段 | 模块 ② 状态机 | 8 选 1 |
| 主行为 | 模块 ③ 行为雷达 | 4 选 1 |
| 参与 | 模块 ④ 参与确认 | 4 选 1 |
| 动作 | 模块 ① 交易卡 | 5 选 1 + 入场价 |
| 失效 | 模块 ① 交易卡 | 单一价位 |

### 星级降级规则

- 参与确认 = "散户推动" → 全局 -2 星
- 参与确认 = "疑似散户" → 全局 -1 星
- 非主力活跃时段（time_heatmap 系数 < 0.5）→ 额外 -1 星
- 阶段 = "无序震荡" → 星级强制 ≤ 2
- 多指标冲突（模块 ②③ 得分差 < 15）→ 额外 -1 星

最终 0~1 星 = "当前不值得交易"，UI 把整个动作区置灰。

## 三、6 个核心模块详解

### ① 交易建议卡片（左上，最关键）

**目标**：输出可直接照做的 5 行指令。

**字段**：
```
方向：做多 / 做空 / 观望
入场区间：[110800, 111150]
止损：110300（跌破即止损）
止盈 T1：111800（R1 阻力位）
止盈 T2：112500（R2 阻力位）
仓位建议：轻仓 / 标仓 / 重仓（基于星级）
失效条件：15min 内未上破 111200 OR 跌破 110300
推理链：
  ・主力吸筹段 Ongoing（avg=110450）
  ・上方 liq_heatmap 5.2x
  ・共振 buy count=4
```

**输入原子**：全部 23 个原子都可能参与，但具体由决策引擎按下文 §六 生成。

**UI 设计**：
- 方向：大字 + 强色（多=绿 / 空=红 / 观望=灰）
- 入场/止损/止盈：右侧数字列，每行一个，点击可直接复制到交易所
- 失效条件：小字灰色
- 推理链：折叠面板，默认展开前 3 条
- 底部有"一键生成 TradingView 标注脚本"（自用工具彩蛋）

**冲突处理**：
- 多空都满足入场条件 → 强制 "观望"
- 入场价在 R1/S1 ±0.2% 内 → 警告 "当前正处于关键位，建议等待明确突破"

### ② 趋势阶段状态机（左中）

**目标**：把市场归类为 8 个阶段之一，决定交易思路。

**8 阶段定义与触发条件**：

| 阶段 | 触发规则（伪代码） | 交易思路 |
|---|---|---|
| 底部吸筹震荡 | `smart_money_cost.Ongoing.type=Accum && 当前价 vs 历史 percentile < 30 && progress < 60 && price 过去 24h 振幅 < 2%` | 左侧挂单做多，不追 |
| 高位派发震荡 | `smart_money_cost.Ongoing.type=Dist && 当前价 percentile > 70 && progress < 60 && 振幅 < 2%` | 左侧挂单做空，不追 |
| 真突破启动 | `破 OB avg_price ±0.3% && cross_exchange_resonance.count >= 3 (近 2 根) && power_imbalance.ratio > 2.5 && 前方 5% 内有 vacuum_band` | 追突破 |
| 趋势延续 | `smart_money_cost.Ongoing 存续 >= 6 根 && avg_price 斜率同向 && trailing_vwap 未被击穿 && 未触发衰竭` | 回踩加仓 |
| 假突破猎杀 | `突破 OB 后 3 根内出现 liquidity_sweep 反向 && 价格回到 OB 内` | 反手做反向 |
| 趋势耗竭 | `trend_exhaustion 连续 >=3 根 > 5 && trend_saturation.progress > 90 && cvd 与 price 背离` | 减仓/反手警报 |
| 黑洞加速 | `price 进入 liq_vacuum 带 && 方向与 smart_money_cost Ongoing 一致 && power_imbalance.ratio > 3` | 顺势加仓 |
| 无序震荡 | 以上都不触发 OR 3 个以上阶段同时触发 | 不交易 |

**状态机机制**：
- 每次 K 线收盘后重算，同时给每个阶段一个 0~100 分数
- 当前阶段 = 最高分 **且** 分数 ≥ 60 的阶段
- 若最高分 < 60 → 强制 "无序震荡"
- 保留**上一阶段**和**次高分阶段**，UI 显示为"来自 X → 当前 Y → 倾向 Z"的三连箭头

**UI 显示**：
```
来自           当前              倾向
底部吸筹  →  真突破启动  →  趋势延续
  62 分        84 分          58 分
```

**冲突处理**：
- 多阶段同时 > 60 → 采用 "无序震荡"
- 阶段在 2 根 K 线内快速切换 → UI 加"状态不稳定"标签

**详细子文档**：[`./modules/state-machine.md`](./modules/state-machine.md)

### ③ 主力行为雷达（左下）

**目标**：告诉用户主力正在做哪个动作，以及每个动作的强度。

**输出结构**（双层）：

**主行为（4 选 1，单选）**：
| 标签 | 触发条件 | 分数计算 |
|---|---|---|
| 吸筹 | `smart_money_cost.Ongoing.type=Accum` | `purity × (1 + 共振 buy 次数) × (1 + CVD 抬升斜率)` |
| 派发 | `smart_money_cost.Ongoing.type=Dist` | 同上反向 |
| 横盘震荡 | 既非吸筹也非派发，或分数都不过阈值 | — |
| 趋势反转 | `liquidity_sweep + CVD 背离 + price 突破上一段 avg_price` | 多条件叠加 |

**叠加警报（0~N 个，多选）**：
| 警报 | 触发条件 | 强度计算 |
|---|---|---|
| 共振爆发 | 近 30min `cross_exchange_resonance.count>=3` 次数 >= 3 | 归一化到 0~100 |
| 诱多 | price 创新高 + fair_value 下行 + CVD 未同步 | 条件满足数 × 35 |
| 诱空 | price 创新低 + fair_value 抬升 + CVD 未同步 | 同上 |
| 衰竭 | trend_exhaustion 近 5 根最大值 | 直接 × 10 |
| 变盘临近 | trend_saturation.progress >= 80 | `(progress - 80) × 5` |
| 护盘中 | price 多次回踩 smart_money_cost.avg_price ±0.3% 且止跌 | 回踩次数 × 20 |
| 压盘中 | price 多次反弹 smart_money_cost.avg_price ±0.3% 且止涨 | 同上 |

**UI 设计**：
```
主行为：吸筹                   ━━━━━━━━━━━  78 分
       （吸筹段已持续 14 根）

叠加警报：
  ⚡ 共振爆发   ━━━━━━━━━━  82
  🔥 变盘临近   ━━━━━━━      64
  🛡️ 护盘中     ━━━━         41
```

**冲突处理**：
- 主行为 = 吸筹 但 诱多 警报 > 70 → 降级为 "吸筹(伴随诱多)"，UI 加问号
- 主行为 = 横盘震荡 但 共振爆发 > 80 → 升级为 "蓄势待发"

**详细子文档**：[`./modules/behavior-radar.md`](./modules/behavior-radar.md)

### ④ 主力参与确认（作为**全局门**，逻辑上先于其它模块）

**目标**：回答"这波到底是不是主力在动"，给其它模块的结论打可信度折扣。

**4 档结论**：

| 结论 | 触发 | 对其它模块的影响 |
|---|---|---|
| **主力真参与** | `共振 count>=3 次数 >=5 (近 1h)` + `power_imbalance 有 ratio>3 事件` + `inst_vol_series 近 10 根合计 > 历史中位数 × 1.5` + `time_heatmap 活跃度 > 1.0` | 全模块满权重 |
| 局部参与 | 以上 4 条件满足 2~3 条 | 全模块 × 0.7 |
| 疑似散户 | 满足 0~1 条 + 价格剧烈波动 | 全模块 × 0.4 + 动作降级为"观望" |
| 垃圾时间 | `time_heatmap.hour` 活跃度 < 0.5 | 强制 "观望"，全模块置灰 |

**UI 设计**：

```
主力参与确认：主力真参与  ✓
  ✓ 跨所共振强度：高（近 1h 7 次）
  ✓ 资金偏离公允：+1.2%
  ✓ 净主动方向：多（连续 12 根）
  ✓ 力量悬殊：极端（ratio 4.8）
  ✓ 活跃时段：亚欧盘交接（系数 1.3）
```

四行勾叉清晰展示，不需要解释。

**为什么放在这么关键的位置**：
- 所有其它模块的结论可信度，取决于这个判断
- 避免"在垃圾时间把噪音当信号"（V1 最容易踩的坑）
- 相当于整个大屏的"安全气囊"

**详细子文档**：[`./modules/participation-gate.md`](./modules/participation-gate.md)

### ⑤ 关键位阶梯图（右上）

**目标**：一眼看清上下各 3 档关键位，从近到远。

**数据源**（5 类候选）：

| 来源 | 字段 | 原始权重 |
|---|---|---|
| smart_money_cost | avg_price | 1.0 |
| trend_price (OB) | avg_price | 0.8 |
| absolute_zones | bottom_price / top_price | 0.7 |
| hvn_nodes | price (Top 10) | 1.0 |
| micro_poc | poc_price | 0.6 |
| liq_heatmap | price | 0.5（不单独成阶梯，但会标注在阶梯旁） |

**筛选算法**：

```
1. 收集所有候选位（上述 5 类）
2. 过滤：只保留距离现价 ±5% 内的
3. 合并：相距 < 0.3% 的位相同类别 → 合并成一个（强度相加）
4. 打分：
   score = source_weight × strength × (1 - decay_pct) × freshness
   - strength = OB volume / HVN rank / zone range 归一化
   - decay_pct = ob_decay 的衰减百分比（若适用）
   - freshness = (now - start_time) 的衰减因子
5. 过滤：分数 < 阈值的丢弃
6. 距离加权：final_score = score / (1 + distance_pct × 10)
7. 上方取 Top 3 → R1/R2/R3（按距离远近排列）
   下方取 Top 3 → S1/S2/S3
8. 要求：相邻两档间距 >= 0.5%，否则跳过
```

**每档显示字段**：

```
R1  111100  ██████    首次测试
    来源：趋势成本带(Accum) + HVN#3
    类型：成本区 + 筹码峰
    强度：强
    适合：反弹反手做空 / 向上破掉可追
```

**UI 设计**：
- 竖直阶梯：R3 在顶 → 现价居中（高亮） → S3 在底
- 每档带一条右侧强度柱（0~100 归一化）
- 颜色：R = 红渐变 / S = 绿渐变
- 点击某一档 → 弹窗显示该位的完整信息 + 在主图高亮

**溢出处理**：
- 超过 R3/S3 的位 → 右上角"查看全部关键位"链接 → 新窗口打开完整表（带搜索筛选）

**详细子文档**：[`./modules/key-levels.md`](./modules/key-levels.md)

### ⑥ 流动性磁吸罗盘（右下）

**目标**：明确"下一次流动性会被扫向哪里"（止盈目标）。

**数据源**：
- liq_heatmap 热度带（主）
- liquidation_fuel 燃料库（辅）

**输出**：上下各 Top 3 磁吸点。

**评分公式**：
```
attraction = intensity × (1 + fuel) / (1 + distance_pct × 5)
```

距离加权系数比关键位阶梯更小 —— 远处的高能磁吸比近处的弱磁吸更值钱（止盈空间问题）。

**UI 设计**：

```
↑ 上方磁吸（空头清算区）
  112500  ████████  5.2x
  113200  █████     3.1x
  114000  ███       2.0x
── 现价 110800 ──
↓ 下方磁吸（多头清算区）
  109200  ███████   4.8x
  108500  ████      2.3x
  107200  ██        1.5x
```

**联动**：
- 交易建议卡片 ①️ 的止盈 T1/T2 直接从磁吸罗盘读取同向 Top 2
- 若上下磁吸比 > 1.5，模块 ② 里触发"方向倾斜"加分

### ⑦ 异动时间线（底部条）

**目标**：让用户对"刚刚发生了啥"有实时感。

**事件源**（过滤后）：
- cross_exchange_resonance 中 count >= 3 的事件
- liquidity_sweep 所有事件
- trend_exhaustion > 5 的 K 线
- power_imbalance.ratio > 3 的 K 线
- 状态机阶段切换
- 主行为标签切换

**UI 设计**：
- 横向滚动条（新事件从右进入）
- 每条：`10:53  ⚡共振 4 所 buy @110850`
- 点击某条 → 主图高亮对应 K 线
- 默认显示最近 30 min，按钮切换 "今日全部"

**重要**：不要把所有指标变动都推进来，只推"有交易意义的异常"，不然变成噪音。

## 四、主图区（中央大图）

**核心原则**：**单一视图**，**不切换 Tab**，所有图层叠加。理由：自用不是展示，不需要分页。

**叠加层（默认全开，可单独隐藏）**：

| 层 | 数据源 | 绘制方式 |
|---|---|---|
| K 线 | Binance/OKX kline | 标准蜡烛 |
| 成本带 | smart_money_cost | 水平条（绿/红，按 purity 调饱和度） |
| 撑压线 | trend_price (OB) | 右侧水平线（粗细 ∝ volume，透明度 ∝ 1-decay） |
| 密集带 | absolute_zones | 半透明矩形 |
| 磁吸带 | liq_heatmap | 右侧渐变条 |
| 血滴 | liquidity_sweep | 价位上的红/绿水滴图标 |
| 共振点 | cross_exchange_resonance (count>=3) | 价位上的圆点（大小 ∝ count） |
| 真空带 | liq_vacuum | 半透明浅色矩形 |
| 动态防线 | trailing_vwap | 绿/红折线 |
| 当前关键位 | 模块 ⑤ 阶梯 | 水平虚线 R3→S3 |

**过滤控制**：
- 图层开关在右上角一个下拉（11 个 checkbox）
- 预设：默认 / 只看关键位 / 只看事件 / 成本带+OB
- **强烈建议保持默认**，因为叠加后的"全景感"正是大屏的价值

## 五、冲突裁决框架（解决指标打架）

这是 V1 最容易翻车的部分。核心做法：**不让任何模块独立说话**，所有结论都走 3 层裁决。

### 层 1：参与度门控（最外层）
```
if 模块④ = 垃圾时间:
    整个大屏置灰 + 强制 "观望"
    return

confidence_multiplier = {
  主力真参与: 1.0,
  局部参与: 0.7,
  疑似散户: 0.4,
}[模块④结论]
```

### 层 2：模块间互校（中间层）
```
if 模块② = 真突破启动 AND 模块③.主行为 != 吸筹/派发:
    模块② -= 20 分

if 模块② = 趋势延续 AND 模块③.警报 有 衰竭 > 70:
    模块② 降级为 "趋势耗竭"

if 模块① 动作 = 追多 AND 模块③.警报 有 诱多 > 60:
    模块① 动作 降级为 "观望" + 警告文字
```

### 层 3：时间一致性（最内层）
```
if 模块 ② 在最近 2 根 K 线内翻转 >= 2 次:
    强制 "无序震荡" + UI 警告 "信号不稳定"

if 某模块分数 < 所有分数的 std dev × 1.5:
    该分数不足以作决策，打问号
```

## 六、决策引擎 — 从原子到行动的完整链路

```
Step 1: 采集 + 存原子
  22 endpoints (实际 18 次请求) → 23 个原子表

Step 2: 模块 ④ 参与确认（门）
  输出：{结论, 可信度 0~1}
  若 = 垃圾时间 → 直接返回 "观望"

Step 3: 模块 ③ 行为雷达
  输出：{主行为, 主行为分数, 警报[]}

Step 4: 模块 ② 状态机
  输入：模块 ③ 结果 + 原子
  输出：{当前阶段, 分数, 上阶段, 下一倾向阶段}

Step 5: 模块 ⑤ 关键位阶梯 + ⑥ 磁吸罗盘
  输出：{R3~S3 列表, 上方磁吸 Top3, 下方磁吸 Top3}

Step 6: 模块 ① 决策
  查决策表（阶段 × 行为 → 动作模板）：
    (底部吸筹震荡, 吸筹)   → 回踩做多 @ S1
    (真突破启动, 吸筹)     → 追多 @ current
    (高位派发震荡, 派发)   → 反弹做空 @ R1
    (假突破猎杀, 反转)     → 反手做反向
    ...
  
  填入具体价位：
    入场 = 查阶梯 ⑤ 对应档
    止损 = 跨过下一档阶梯
    止盈 T1/T2 = 磁吸罗盘 ⑥ Top 2
  
  星级 = phase_score × participation_multiplier × behavior_score / 10000 → 1~5

Step 7: 构造结论带文案
  渲染到顶部 Hero Strip
```

## 七、UX 原则与视觉规范

### 色彩
```
多头     #00d68f (亮绿)
空头     #ff3d71 (亮红)
观望/中性 #8f9bb3 (灰蓝)
警告     #ffaa00 (橙)
背景     #151a30 (深夜蓝)
卡片底   #1f2540 (浅一级)
字 主     #e4e9f2 (米白)
字 次     #8f9bb3 (灰蓝)
```

**重要**：K 线和指标叠加层的颜色要和结论色**分开用**。比如结论红绿用的是鲜艳色，K 线涨跌用的是暗一级（避免眼花）。

### 字号梯度
- 结论带标题 `48px`
- 模块主数据 `32~40px`
- 模块小标签 `14~16px`
- 时间线 `12px`
- 极小字（字段标注）`10px`

### 交互
- **滑动时间**：主图拖拽时，所有模块**不响应**（避免误操作）。需显式点击"回放历史时刻"按钮进入回放模式
- **点击联动**：任何模块的一个位点击 → 主图对应时间高亮 + 该位投影到右侧阶梯
- **长列表策略**：任何列表超过 8 项 → 底部 "查看全部 (N)" 链接 → 新窗口打开完整页（不是 tab）
- **键盘快捷键**：空格 = 暂停实时 / R = 回到实时 / 1~8 = 切换模块展开

### 防误判设计
- "当前不值得交易" 状态下，整屏加浅灰遮罩（提醒这是 "不做" 状态）
- 低星级动作，"入场"按钮置灰不可点击（若后续做一键下单）
- 信号在 2 根 K 线内反转 > 1 次 → 所有卡片右上角出现 ⚠️ "信号不稳"

## 八、技术栈建议（非强制）

### 前端
- **框架**：React + TypeScript
- **图表**：lightweight-charts（TradingView 开源）— K 线 + 叠加层原生支持
- **状态**：Zustand 或 Jotai（避免 Redux 的臃肿）
- **实时**：WebSocket 订阅 → 推送原子更新，前端驱动所有模块重算

### 后端
- **语言**：Node.js / Python（Python 更适合指标计算，但 Node 前后端统一 TS）
- **存储**：SQLite（单机自用足够）+ 日级分区备份
- **调度**：cron + node-cron，按 K 线收盘时间对齐
- **WebSocket**：主动 push 新原子到前端，前端收到后触发模块重算

### 自测
- 所有模块都是纯函数 `(atoms[]) => ModuleOutput`，便于单元测试
- 回放模式：传历史某个时刻的原子快照进去，所有模块应输出当时的结论 — 这就是天然的"回测"基础

## 十、我的几个担忧与建议（必须说）

### 担忧 1：阶段状态机会误判
8 个阶段的触发条件是靠"经验阈值"设定的，**没有历史数据回测的前提下，阈值几乎肯定会偏**。

**建议**：
- 所有阈值做成**可配置参数**（配置文件 or UI 滑块），不写死在代码里
- 先跑 2~4 周真实数据，观察状态机输出 vs 行情走势，手动调参
- 不要一上来就全 8 阶段，先上 3~4 个最典型的，其它暂归为 "无序震荡"

### 担忧 2：决策查找表覆盖不全
32 种组合我只给了 10 种典型，剩下 22 种怎么办？

**建议**：
- 未列出的组合 → 默认 "观望"（安全兜底）
- UI 上标注 "未覆盖组合，建议观望"
- 记录日志，事后人工补充决策表

### 担忧 3：UI 小白能看懂吗？
这个设计其实**不那么小白**。22 个指标压成 7 个模块，但每个模块里还是有"吸筹 78 / 共振 82"这种分数。真正小白可能只能看懂顶部结论带。

**建议**：
- V1 的"小白模式"：只保留顶部结论带 + 主图 + 阶梯图，隐藏其它模块
- UI 上提供"模式切换"按钮：Pro / Lite
- 你自己肯定用 Pro，但演示给别人看可以切 Lite

### 担忧 4：主图叠加 10 层会糊
10 个叠加层全开，K 线会淹没在里面。

**建议**：
- 默认只开 5 层（K 线 / 成本带 / 撑压线 / 血滴 / 共振点）
- 其它层默认关闭，点击右上角按需开启
- 右侧关键位阶梯已经把"关键位信息"从主图抽出来了，主图不必负担

### 担忧 5：模块太多，视线跳跃
屏幕上同时有 7 个模块，人眼会无所适从。

**建议**：
- 只聚焦一个"视觉中心"：**结论带**。其它模块都是"辅助证据"
- 用视觉层级（字号大小、留白、高亮色）强制引导视线路径：
  结论带 → 主图 → 关键位阶梯 → 行为雷达 → 其它
- 首次打开大屏，给个 15 秒的 onboarding 动画，点亮每个区的用途
